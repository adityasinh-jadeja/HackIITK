To see the application running, you must start the backend, and then the Electron shell:

**Terminal 1 (Backend):**
```powershell
cd backend-python
.\venv\Scripts\Activate
python -m uvicorn app.main:app --port 8000 --reload
```

**Terminal 2 (Electron Shell + React):**
```powershell
cd frontend
npm run dev
# (Keep this running in the background)
```

**Terminal 3 (Electron Desktop App):**
```powershell
cd electron-shell
npm start
```

## Phase 2: DOM Scanner & Core Security Engine
**Status:** ✅ COMPLETED

### 📌 What Was Completed
- Integrated **Playwright** (`sync_playwright` via `asyncio.to_thread` to avoid Windows event loop conflicts) into the Python backend (`page_renderer.py`) for headless DOM extraction and JS execution.
- Built the **DOM Scanner Engine** (`dom_scanner.py`) utilizing BeautifulSoup to parse and evaluate the exact rendered DOM against 5 specialized attack vectors:
  1. Prompt injection via visible text
  2. Hidden instructions via CSS (display:none, opacity:0, font-size:0, off-screen positioning)
  3. Deceptive buttons/forms (hidden JWTs, Javascript actions, suspicious inputs)
  4. JavaScript-based dynamic content injection (`eval`, `setTimeout`, event handlers)
  5. Phishing login pages (Brand impersonation, fake SSL badges, urgency language)
- Created the `/api/scan` REST endpoint to manually trigger Playwright page loading, execute the DOM scanner, persist threats to MongoDB, and broadcast `SCAN_STARTED` and `SCAN_COMPLETE` events via WebSocket.
- Fixed a WebSocket broadcast double-nesting bug (`{"data": {"data": ...}}`) in `handler.py` so real-time push updates function correctly.
- Enhanced the **Dashboard UI** (`Dashboard.jsx`) to consume live WebSocket updates, enabling real-time render of Threat Cards containing severity badges, confidence scores, XPaths, and collapsible raw HTML snippets.
- Validated all 6 mock test HTML pages (`prompt_injection.html`, `hidden_css.html`, `benign_shopping.html`, etc.) to ensure reliable detection of attack vectors and guarantee 0% false positives on benign pages.

### 🚀 How to Run the Servers
*(Same as Phase 1)* Follow the startup commands from Phase 1 to boot the Backend, Frontend, and Electron Shell in three separate terminals.
